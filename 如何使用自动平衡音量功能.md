# 如何使用自动平衡音量功能

## 功能位置
自动平衡音量功能位于音频播放器的**设置弹窗**中。

## 访问步骤

### 1. 打开应用
- 启动HarmonyOS音频播放器应用
- 进入主播放界面

### 2. 找到设置按钮
在播放界面的右上角，你会看到一个**齿轮图标**或**滑块图标**的按钮：
```
[返回按钮] [歌曲信息] [设置按钮]
```

### 3. 打开设置弹窗
- 点击右上角的**设置按钮**
- 会从底部弹出一个设置面板

### 4. 找到自动平衡音量控制
在设置面板中，向下滚动可以看到以下控制项：

1. **禁用**开关 - 禁用所有音量控制（现有功能）
2. **音频流音量**控制（现有功能）
3. **自动平衡音量**开关 - 新添加的功能
   - 默认状态：**启用**（绿色）
   - 点击开关可以启用/禁用自动平衡功能

4. **压缩强度**滑块 - 仅在自动平衡启用时显示
   - 范围：10% - 100%
   - 默认值：70%
   - 拖动滑块调整压缩强度
   - 右侧显示当前百分比数值

### 5. 功能说明
在滑块上方有一段说明文字：
> "自动平衡音量功能可以平滑音频音量，避免音量突变。压缩强度越高，音量变化越小。"

## 功能效果

### 启用自动平衡时：
- 音频音量会被平滑处理
- 突然的音量变化会被压缩
- 压缩强度越高，音量变化越小
- 适合播放音量差异大的音频内容

### 禁用自动平衡时：
- 使用原始音频音量
- 保持音频的原始动态范围
- 适合需要保持音频原始特性的场景

## 技术实现

### 音量压缩算法
算法位于 `AudioVolumeController.ets` 中的 `applyVolumeCompression()` 方法：
```typescript
// 压缩公式：compressedVolume = Math.pow(volume, compressionRatio)
// compressionRatio 范围：0.1 - 1.0
// 值越小压缩越强，音量变化越小
```

### 实时应用
- 在 `AudioRendererController.ets` 的 `writeAudioData()` 方法中调用
- 每次写入音频数据时都会应用压缩
- 根据用户设置的压缩强度动态调整

### 状态持久化
- 使用 AppStorage 存储用户设置
- 开关状态和压缩强度值会被保存
- 应用重启后设置保持不变

## 测试建议

### 测试场景1：音量突变测试
1. 播放一段有音量突变的音频（如电影预告片）
2. 启用自动平衡功能
3. 调整压缩强度，观察音量变化
4. 比较启用/禁用时的效果差异

### 测试场景2：音乐播放测试
1. 播放一首动态范围较大的音乐
2. 启用自动平衡，设置不同压缩强度
3. 感受不同设置下的听觉体验

### 测试场景3：状态持久化测试
1. 调整设置（启用自动平衡，设置压缩强度为50%）
2. 退出应用
3. 重新启动应用
4. 验证设置是否保持

## 注意事项
1. 压缩强度设置是实时生效的，无需重启应用
2. 自动平衡功能只影响音频播放，不影响系统音量
3. 压缩算法设计为平滑处理，不会造成音频失真
4. 在低压缩强度（接近10%）时，音量变化会更明显
5. 在高压缩强度（接近100%）时，音量变化会很小

# 自动平衡音量功能测试指南

## 功能概述
已成功实现音频播放器的自动平衡音量功能，该功能可以：
1. 平滑音频音量，避免音量突变
2. 提供可调节的压缩强度（10%-100%）
3. 实时应用音量压缩算法

## 测试步骤

### 1. 启动应用
- 运行HarmonyOS应用
- 进入音频播放器界面

### 2. 找到自动平衡音量控制
- 在播放器设置中查找"自动平衡音量"开关
- 开关默认状态：启用（绿色）
- 压缩强度滑块默认值：70%

### 3. 测试功能
#### 3.1 启用/禁用自动平衡
- 点击开关可以启用/禁用自动平衡功能
- 启用时：音量将被平滑处理
- 禁用时：使用原始音量

#### 3.2 调整压缩强度
- 拖动滑块调整压缩强度
- 范围：10% - 100%
- 步长：5%
- 数值实时显示在滑块右侧

#### 3.3 观察效果
- 播放音频时，启用自动平衡功能
- 观察音量是否更平稳
- 调整压缩强度，感受不同强度下的效果

### 4. 技术验证
#### 4.1 状态持久化
- 退出应用后重新启动
- 验证设置是否保存
- 自动平衡开关状态应保持不变
- 压缩强度值应保持不变

#### 4.2 音量压缩算法
- 算法位于 `AudioVolumeController.ets`
- 使用公式：`compressedVolume = Math.pow(volume, compressionRatio)`
- 压缩比例越高（接近1.0），音量变化越小
- 压缩比例越低（接近0.1），音量变化越大

#### 4.3 实时应用
- 算法在 `AudioRendererController.ets` 中实时应用
- 每次写入音频数据时都会应用压缩
- 确保音频播放流畅无卡顿

## 预期效果
1. **音量平滑**：音频中突然的音量变化被平滑处理
2. **可调节性**：用户可以根据需要调整压缩强度
3. **实时响应**：设置更改立即生效
4. **状态保持**：用户设置被持久化保存

## 故障排除
1. **滑块不显示**：检查自动平衡开关是否启用
2. **音量无变化**：确认压缩强度设置是否有效
3. **编译错误**：检查AppStorage初始化是否正确

## 文件修改清单
1. `CommonConstants.ets` - 添加常量定义
2. `PlayerViewModel.ets` - 添加AppStorage状态
3. `AudioVolumeController.ets` - 实现音量压缩算法
4. `AudioRendererController.ets` - 应用音量压缩
5. `Player.ets` - 添加UI控件和状态管理
6. `string.json` (base/zh_CN/en_US) - 添加多语言字符串

## 技术要点
- **压缩算法**：`compressedVolume = Math.pow(volume, compressionRatio)`
- **状态管理**：使用AppStorage实现跨组件状态共享
- **条件渲染**：仅在启用自动平衡时显示压缩强度滑块
- **类型安全**：使用类型断言确保AppStorage值类型正确
